auto-core.log

Main entrypoint to auto core logging, defines basic levels that we use to log.

Logical order of logs is: trace -> debug -> info -> warn -> error -> fatal

Design decision: * the log is relying on configuration namespace, so nothing will be logged before that * the log api is hosted in a cljc file so log is accessible to namespaces in cljc * Rationle: * deciding to push code in cljc should not prevent to log * postponing the decision to host code in frontend or backend is an objective * Consequence: * This namespace is the entrypoint for both clj and cljs implementations
